AWS Integrated GRC Platform

Governance, Risk, and Compliance Capstone Project by Joy Anunwa

📌 Project Overview

The AWS Integrated GRC Platform is a comprehensive Governance, Risk, and Compliance solution that demonstrates how to build, deploy, and manage an enterprise-grade GRC platform using Amazon Web Services.

This project integrates 13 AWS services to provide continuous compliance monitoring, risk assessment, audit logging, and real-time dashboards for six major compliance frameworks including ISO 27001, NIST CSF, PCI DSS, HIPAA, GDPR, and SOC 2.

⚠️ Problem Statement

Organizations today face increasing complexity in managing multiple compliance frameworks simultaneously. Traditional manual approaches using spreadsheets and emails have serious limitations:

  • Manual and time-consuming - Someone must physically check if controls are implemented
  • Reactive, not proactive - Problems are discovered after they've happened
  • No central visibility - Different teams use different spreadsheets
  • Difficult to prove to auditors - Scrambling for evidence during audits

🎯 Project Objectives

  • Design scalable cloud architectures for GRC applications
  • Implement AWS native security and compliance services
  • Write Infrastructure as Code using CloudFormation
  • Develop serverless applications with AWS Lambda
  • Implement comprehensive audit logging and monitoring
  • Map technical controls to compliance frameworks
  • Create automated compliance checking and reporting

🏛️ System Architecture

The platform uses a multi-tier architecture with public and private subnets:

  • Public Subnets - Host the Application Load Balancer (internet-facing)
  • Private Subnets - Host the RDS database, ECS containers, and Lambda functions
  • VPC - Isolated network with CIDR 10.0.0.0/16 across two availability zones
  • Security Groups - Act as virtual firewalls controlling traffic

📋 Architecture Design

The platform follows a multi-tier architecture with the following components:

  • Presentation Layer: React dashboard hosted on CloudFront with S3
  • Application Layer: ECS Fargate containers running the GRC application
  • Data Layer: RDS MySQL for relational data, DynamoDB for real-time status
  • Security Layer: AWS Config, CloudTrail, IAM, KMS encryption
  • Monitoring Layer: CloudWatch metrics, alarms, and SNS notifications

💻 Technology Stack

  • Infrastructure: AWS CloudFormation, VPC, IAM, KMS
  • Compute: AWS Lambda (Python 3.11), ECS Fargate
  • Storage: RDS MySQL, S3, DynamoDB
  • Security & Compliance: AWS Config, CloudTrail, Security Hub
  • Monitoring: CloudWatch, SNS

🔒 Security Design

  • Network Security: VPC with public/private subnet isolation, security groups, NAT Gateway
  • Data Security: Encryption at rest (RDS, S3, DynamoDB), encryption in transit (TLS)
  • Access Control: IAM roles with least privilege, MFA enabled
  • Audit & Monitoring: CloudTrail for API logging, CloudWatch for metrics and alerts

🌐 Network Stack (VPC)

The CloudFormation network stack creates:

  • VPC with CIDR 10.0.0.0/16
  • 2 public subnets (for load balancer)
  • 2 private subnets (for database and ECS)
  • Internet Gateway and NAT Gateway
  • Route tables for public and private traffic
  • Security groups for ALB, ECS, and RDS

🗄️ Database Stack (RDS)

The CloudFormation database stack creates:

  • RDS MySQL 8.0 instance (db.t3.micro, 20GB storage)
  • S3 buckets for evidence and compliance reports
  • DynamoDB tables for compliance status, risk register, and controls
  • KMS keys for encryption

🖥️ EC2 Bastion Host

An EC2 bastion host is deployed in a public subnet to provide secure SSH access to resources in private subnets:

  • Purpose: Secure jump server to access RDS database
  • Location: Public subnet
  • Access: SSH key pair authentication
  • Security: Security group allows SSH only from specific IPs

📦 S3 Buckets

S3 buckets store all compliance evidence and logs:

  • Evidence Bucket: Stores compliance evidence documents with versioning enabled
  • Compliance Reports Bucket: Stores generated compliance reports
  • Config Logs Bucket: Stores AWS Config logs
  • CloudTrail Logs Bucket: Stores audit logs for long-term retention

⚡ Lambda Function

The compliance monitor Lambda function (Python 3.11) runs on a schedule and:

  • Queries AWS Config for compliance status
  • Calculates compliance percentages
  • Stores results in DynamoDB
  • Generates compliance reports in S3
  • Sends SNS alerts when compliance drops below thresholds

📊 CloudWatch Alarm

CloudWatch alarms monitor the Lambda function for errors:

  • Alarm Name: grc-lambda-errors
  • Metric: Lambda Errors
  • Threshold: 1 error
  • Action: Send SNS notification to grc-compliance-alerts
  • Status: OK (No errors detected)

📋 AWS Config & CloudTrail

  • AWS Config - Continuously monitors resource configurations and evaluates them against compliance rules
  • CloudTrail - Records every API call for audit purposes, creating an immutable audit trail

📊 Dashboard Deployment

The GRC dashboard is a React-based web application hosted on CloudFront with S3. It displays:

  • KPI cards (average compliance, critical risks, active frameworks)
  • Compliance status for each framework with progress bars
  • Risk register with severity levels and scores
  • Control implementation status

📜 Compliance Frameworks Supported

  • ISO 27001:2022 - International information security standard (114 controls)
  • NIST CSF - US government cybersecurity framework
  • PCI DSS 3.2.1 - Payment card industry standard
  • HIPAA - Healthcare data protection
  • GDPR - EU data privacy regulation
  • SOC 2 - Service organization controls

🏆 Key Achievements

  • ✅ 13 AWS services integrated into a working GRC platform
  • ✅ 6 compliance frameworks supported
  • ✅ 22/22 comprehensive tests passing
  • ✅ Complete Infrastructure as Code using CloudFormation
  • ✅ Automated compliance monitoring with Lambda
  • ✅ Live dashboard accessible via CloudFront
  • ✅ Audit logging with CloudTrail

🔧 Challenges & Solutions

  • Challenge: CloudFormation S3 encryption syntax error
    Solution: Fixed by wrapping ServerSideEncryptionConfiguration inside BucketEncryption
  • Challenge: KMS key alias property error
    Solution: Changed KeyId to TargetKeyId
  • Challenge: RDS connection timeout from local machine
    Solution: Used EC2 bastion host in same VPC for secure access
  • Challenge: Free Tier backup retention limits
    Solution: Set BackupRetentionPeriod to 1 day

💡 Lessons Learned

  • Infrastructure as Code is powerful - CloudFormation made redeployment easy after failures
  • Free Tier has limitations - Backup retention and encryption features are restricted
  • Security groups are critical - Proper rules are essential for connectivity
  • Always use IAM users - Never use root account for daily operations
  • Test early, test often - 22 tests caught issues before presentation

📚 Conclusion

This project successfully demonstrates an enterprise-grade GRC platform on AWS that provides automation, real-time visibility, audit trails, and cost savings. The platform is fully deployed, tested, and documented, ready for presentation.
